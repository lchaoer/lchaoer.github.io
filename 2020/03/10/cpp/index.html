语言基础

Struct和Class的区别？

  • struct默认限定符为public；class默认限定符为private。

static的用法？

https://www.zhihu.com/question/274217344

  • 内部链接的全局函数/变量声明使用static修饰，表明该函数/变量只在所在的编译单元内可见。与之相关的是外部链接声明，使用extern修饰。
  • 静态局部变量在程序第一次运行到该声明时做初始化，作用域结束后变量不销毁，下一次运行时仍保持状态。
  • 静态成员函数/变量表示函数/变量不绑定至实例。静态变量除了声明外还需要在类外显示定义。

#define和const的区别？

  • 宏定义发生在编译前的预处理阶段，直接进行文本替换；const发生在编译阶段。
  • 宏定义没有分配空间，只是对相关数据进行插入替换，多次替换需要的数据量大；const定义的变量只会分配一次内存空间，在后续多次使用不会多次复制。
  • 宏定义不检查类型；const会检查数据类型。

malloc和new的区别？

  • malloc是标准库函数；new是运算符。
  • malloc仅仅分配空间；new的实现有用到malloc，除了分配空间外，还会调用构造函数。
  • malloc返回的是void*指针；new返回的是具体类型的指针。

指针和引用的区别？

  • 引用是变量的别名，不占具体的存储空间；指针是具体的变量，占存储空间。
  • 引用在声明时必须初始化为另一变量，且初始化后不可改变；指针声明时可不用初始化，且可指向新的变量。
  • 引用不可为空；指针可为空指针。

volatile的作用？

  • volatile定义变量的值是易变的，每次用到这个变量的值的时候都要去带内存地址重新读取这个变量的值，而不是读寄存器内的缓存。
  • 多线程中被几个任务共享的变量需要定义为volatile类型。

常量指针和指针常量？

  • 常量指针是指向一个常量的指针，只读。const int *p
  • 指针常量是不能改变指向的指针。int *const p

数组名和指针的区别？

  • 二者均可以通过增减偏移量来访问相应地址中的元素。
  • 数组名不是真正意义上的指针，可以理解为指针常量，无法进行自增++操作。
  • 当数组名作形参传递给函数后，就退化成一般指针，可进行自增操作，但sizeof无法获得数组大小。

字节对齐？

https://zhuanlan.zhihu.com/p/30007037

现代计算机中内存空间都是按照 byte 划分的，从理论上讲似乎对任何类型的变量的访问可以从任何地址开始，但是实际的计算机系统对基本类型数据在内存中存放的位置有限制，它们会要求这些数据的首地址的值是某个数k（通常它为4或8）的倍数，这就是所谓的内存对齐。

  • 假如没有内存对齐机制，数据可以任意存放，现在一个int变量存放在从地址1开始的连续四个字节地址中，处理器去取数据时，要先从0地址开始读取第一个4字节块,剔除不想要的字节（0）,然后从地址4开始读取下一个4字节块，同样剔除不要的数据（5，6，7），最后留下的两块数据合并放入寄存器。这需要做很多工作。
  • 现在有了内存对齐，int类型数据只能存放在按照对齐规则的内存中，比如说0地址开始的内存。那么现在该处理器在取数据时一次性就能将数据读出来了，而且不需要做额外的操作，提高了效率。
  • 可以通过预编译命令#pragma pack(n)，n = 1,2,4,8,16来改变每个编译器的对齐单位。

内存对齐的规则：

有效对其值：是给定值#pragma pack(n)和结构体中最长数据类型长度中较小的那个。

  1. 结构体第一个成员的偏移量（offset）为0，以后每个成员相对于结构体首地址的 offset 都是该成员大小与有效对齐值中较小那个的整数倍，如有需要编译器会在成员之间加上填充字节。
  2. 结构体的总大小为有效对齐值的整数倍，如有需要编译器会在最末一个成员之后加上填充字节。

大端小端？如何判断？

大端：数据的高字节保存在低地址中。

小端：数据的低字节保存在低地址中。

面向对象

虚函数、纯虚函数、虚函数表？

https://zhuanlan.zhihu.com/p/54145222

https://zhuanlan.zhihu.com/p/75172640

  1. 纯虚函数
    • 若基本的某个方法不需要实现，但希望派生类能够重写该函数，需要定义虚函数。
    • 相当于一个接口，含纯虚函数的类称为抽象类，无法被实例化。
  2. 虚函数
    • 虚函数是C++多态性的体现，目的是为了当基类指针或引用指向派生类对象时，能够自动调用相应的、和基类同名的虚函数。（能够调用派生类，以及派生类中和基类对应的虚函数）。当某个虚函数通过指针或者引用调用时，编译器产生的代码直到运行时才能确定到底调用哪个版本的函数。被调用的函数是与绑定到指针或者引用上的对象的动态类型相匹配的那个。因此，借助虚函数，我们可以实现多态性。
    • 析构函数最好使用虚函数。否则删除一个实际指向派生类的基类指针，只会调用基类的析构函数，而不会调用派生类的析构函数以及派生类数据成员的析构函数。这样就可能造成内存泄露。
  3. 虚函数表
    • 虚表是一个函数查询表，以动态捆绑的方式解析函数调用。每个具有虚函数的类都有一张虚表，这个表是在编译阶段建立的静态数组，其中包含了每个虚方法的函数指针（这些指针指向的是该类可见的派生最远的函数实现）。
    • 虚表是属于类的，而不是属于某个具体的对象，一个类只需要一个虚表即可。同一个类的所有对象都使用同一个虚表。
    • 为了指定对象的虚表，对象内部包含一个虚表的指针，来指向自己所使用的虚表，这里我们称为*__vptr。当类的实例被创建时，这个指针指向该类所对应的虚表。这样，当使用某个对象调用虚方法时，通过该指针查找虚表，然后根据实际的对象类型执行正确版本的方法调用。
    • 当派生类继承了基类的时候也会继承其虚函数表，当派生类重写基类中虚函数时候，会将其继承到的虚函数表中的地址替换为重新写的函数地址。

进阶用法

C++的四种类型转换？

  1. static_cast
    • 基本数据类型之间的转换
    • 上行转换，将派生类的指针或引用转化为基类表示（下行转换不安全）
    • 把void指针转换成目标类型的指针（不安全）
  2. const_cast
    • 消除指针或引用的const或volatile
  3. dynamic_cast
    • 其他三类都是编译时完成的，这个是运行时才处理，会进行类型检查
    • 必须是指针或引用，成功返回指针，失败返回NULL
    • 支持上行转换，下行转换，会进行类型检查（安全）
    • 基类中一定要有虚函数，否则无法通过编译（因为虚函数说明可能有基类指针指向派生类对象），运行时类型信息存储在虚函数表中
  4. reinterpret_cast
    • 根据原数据的二进制形式进行转换。可以转换成任何类型的指针（不安全）

智能指针

https://zhuanlan.zhihu.com/p/54078587

当使用指针进行动态分配内存时，容易出现错误。

即若忘记释放动态分配的内存的话，会造成内存泄漏；若多次分配或释放内存时，可能造成重复释放等错误。

  • 栈上的变量比堆上的变量要安全，因为当栈上的变量离开作用域后，会自动销毁。智能指针结合了栈的安全性和堆的灵活性。
  1. shared_ptr
    • 该指针使用引用计数，来跟踪指向同一个对象的指针数。即可以由多个指针指向同一块动态分配的内存。当最后一个指针离开时，才释放该动态分配的内存。
  2. unique_ptr
    • 该指针指向的对象不允许多个指针共享，只有一个指针有内存所有权。
  3. weak_ptr
    • 使用shared_ptr可能会导致“循环引用”的问题，而weak_ptr可以解决该问题。
    • 该指针不会修改对象的引用计数，这意味着它不会对内存进行管理，仅仅只能访问。但它能检测所指向的对象是否已经被释放，从而避免了非法内存访问。

右值引用、移动语义、move？

https://zhuanlan.zhihu.com/p/54050093

https://blog.csdn.net/li1914309758/article/details/81663488

右值引用是C++11引入的一个非常重要的技术，因为它是移动语义（Move semantics）与完美转发（Perfect forwarding）的基石：

  • 移动语义：将内存的所有权从一个对象转移到另外一个对象，高效的移动用来替换效率低下的复制，对象的移动语义需要实现移动构造函数（move constructor）和移动赋值运算符（move assignment operator）。
  • 完美转发：定义一个函数模板，该函数模板可以接收任意类型参数，然后将参数转发给其它目标函数，且保证目标函数接受的参数其类型与传递给模板函数的类型相同。

左值：可以将左值看成是一个可以获取地址的量。（可以用来标识一个对象或函数）

右值：纯粹的数值字面量或可以被重用的临时对象。

左值引用与右值引用：

  1. 非const左值引用只能绑定到非const左值；
  2. const左值引用可绑定到所有左右值；
  3. 非const右值引用只能绑定到非const右值；
  4. const右值引用可绑定到const右值和非const右值。

移动语义

例：DynamicArray<int> b = a(10);

  • 当未使用移动语义（移动构造函数）时，若进行一个对象的赋值，会首先使用构造函数生成一个临时对象，接着再将临时对象拷贝过来。整个过程需要执行2次构造函数，2次析构函数。
  • 若使用移动语义（移动构造函数），则会直接将对象a的所有权直接转移到对象b中，整个过程只需要执行1次构造函数，2次析构函数。
  • 用移动而不是复制来避免无必要的资源浪费，从而提升程序的运行效率。
