计算机网络课程复习

概述

计算机网络就是互连的、自治的计算机集合。

  • 网络核心：路由器或分组转发设备。
  • 网络边缘：主机、网络应用。

网络协议：是为进行网络中的数据交换而建立的规则、标准或约定。

接入网络：数字用户线路DSL、电缆网络。

  • 带宽：单位时间内通过链路的数据量，通常以bps表示（比特/秒）。

数据交换类型：

  1. 分组交换：报文分拆出来的一系列相对较小的数据包，分组交换需要报文的拆分与重组。采用存储-转发的交换方式。
    • 传输时延=$L/R$，L表示分组大小，R表示传输速率。
  2. 电路交换：端到端，通信前需建立电路。多路复用，频分FDM、时分TDM。
  • 分组交换允许更多用户同时使用网络。分组交换适用于突发网络，可能产生拥塞，分组延迟和丢失。

ISP：因特网服务提供商。

  • 端系统通过接入ISP连接到Internet

时延、丢包、吞吐量。

时延：

  • 结点处理时延。
  • 排队时延。
  • 传输时延。$L/R$，L表示分组长度，R表示链路带宽。
  • 传播时延。$d/s$，d表示物理链路长度，s表示信号传播速度。

丢包：

  • 丢包率=丢包数/已发送分组总数

吞吐量：

  • 表示发送端与接收端之间的传送数据速率
  • 瓶颈链路：限制端到端吞吐量的链路。

协议层次和服务类型

计算机网络体系结构是分层结构。

因特网的分成结构：

  • 应用层：支持网络应用
  • 传输层：进程间的数据传输
  • 网络层：数据报从源与目的之间的路由，数据分组与转发
  • 链路层：相邻网络元素（主机、交换机等）的数据传输
  • 物理层：比特流

OSI参考模型：

  • 应用层、表示层、会话层、传输层、网络层、链路层、物理层

应用层

将应用之间需要通信的数据进行封装。

常见的应用层协议：HTTP、FTP、DNS、SMTP、POP3、IMAP

网络结构

  1. P2P
  2. client-server

不同网络的进程通信，使用socket进行交换报文。ip地址+端口号。

进程通信时，使用TCP/UDP协议进行数据的传输。

Web和HTTP

统一资源定位符URL：www.someschool.edu/someDept/pic.gif

（host name+path name，其中edu后为path，edu及以前为host）

HTTP：超文本传输协议，用于web应用之间的数据传输，client-server模型。

HTTP是一个无状态的协议，也就是说服务器不会保存关于客户端的历史请求信息。

HTTP的通信基于TCP，每一次通信时都要使用TCP建立三次握手，第三次可携带信息；在结束通信时需要进行TCP的四次挥手。通信时可以建立持久性连接或非持久性连接。

RTT：Round-Trip Time，往返时延。数据从一段传送到另一端再返回的时延。

对于非持久性连接的HTTP协议，响应时间=2RTT+file transmission time。

  • 请求响应的报文格式
  • 响应状态码
  • 请求方法

Cookie

  • cookie首部行，存在于HTTP请求和响应报文中
  • 浏览器存储用户的cookie信息
  • 服务器后台存储对应用户的cookie信息

Web caches：可以使客户端不用访问数据对应的原始服务器，缓解服务器的请求压力。

条件get：当web caches的数据长时间没有更新时，不返回数据。

电子邮件相关协议

发送电子邮件由三部分组成

  1. user agent
  2. mail servers
  3. protocol

在发送电子邮件时，一般由用户传到自己对应的mail server，然后再由mail server传到目的mail server，最后别人查看邮件时，需要查看他们自己对应的mail server来获取邮件信息。

发送邮件基于的传输协议：TCP。

SMTP：发送协议。持久性连接，发送信息必须为7位ASCII码。端口号25

POP3、IMAP：接受访问协议。区别如下

  • POP3：允许电子邮件客户端下载服务器上的邮件，但是在客户端的操作（如移动邮件、标记已读等），不会反馈到服务器上。
  • IMAP：提供webmail 与电子邮件客户端之间的双向通信，客户端的操作都会反馈到服务器上，对邮件进行的操作，服务器上的邮件也会做相应的动作。

DNS

DNS：域名管理系统，实现对域名和ip地址的映射关系。

由于DNS实现负载均衡，其一个域名可能对应多个ip地址，在查询DNS服务器时进行动态调整，返回对应ip地址。

DNS由分布式数据库实现，从上到下分别对应根域名、顶级域名、权威域名、本地域名等

查询方式：迭代查询、递归查询。

若一个域名从未缓存过，DNS会在查询后进行缓存。

DNS记录：resource records（RR）。

  • RR format：（name, value, type, ttl）
  • type=A：name=host name, value=ip address
  • type=NS：name=domain, value=hostname of authoritative name server for this domain
  • type=MX：value=name of mailserver associated with name
  • type=CNAME：name=name for some “canonical” (the real) name, value=canonical name

网络层

将数据报从发送端传送到接收端。

网络层与路由器

网络层由数据平面和控制平面组成。数据平面负责控制对数据的转发，控制平面用于控制计算报文之间要如何进行传输，才能从发送端到接收端。

两大功能

  • 转发：将分组从路由器的输入端口转移到合适的输出端口。
  • 路由选择：确定分组从源到目的经过的路径。

控制平面的两种实现方式：

  1. 一个远程控制器直接控制多个路由器，进行统一路由选择，然后将转发表下发到各个路由器。
  2. 每个路由器独立地控制自己，维护自己的转发表，多个路由器之间能够进行相互交互。

网络服务模型：定义了分组在发送与接收端系统之间的端到端运输特性。具有确保分组到达目的地、有序分组交付等功能。

路由器的组成：物理层、数据链路层、网络层。

在选择路由器的端口时，满足最长前缀匹配原则。

（将报文从输入端转发向合适的输出端）交换结构的类型：

  • 内存 memory
  • 总线 bus
  • 互联网络 interconnection network

输入、输出端口排队（缓存、调度策略）

IP协议

ip报文

报文格式。

MTU：最大传输单元。

每个网络链路有各自的MTU，当报文段过大时，会将报文进行分片（fragmentation），只有在最后的目的主机才会进行重组（reassembly）

ipv4

ipv4地址有32位，用点分十进制表示。其地址的接口存在于每个主机和路由器，由ip地址将主机和路由器联系起来。

ip地址的表示：

  • 高比特位：子网

  • 低比特位：主机

  • 223.1.3.0/24 ：表示前24位为子网，24为子网掩码。

CIDR：无类别域间路由。（子网部分无固定位数）

DHCP：动态主机配置协议。（自动获得ip地址、默认网关、子网掩码等）

ISP：互联网服务提供商。（提供一大组IP地址）

路由聚合：转发往一个子网，而不是单个IP。

最长前缀匹配原则。

NAT：网络地址转换。在本地局域网内用局域网的ip地址，若要接入全球互联网，则与路由器通信时会分配一个全球的ip地址。

ipv6

  • 报文首部固定40个字节。
  • 在传输过程中不可对数据报进行分片。

以上目的：加快数据报处理和转发的速度。

报文格式。

ipv4转ipv6

  • 隧道tunneling
  • 把ipv6封装到ipv4中

网络层控制平面

路由算法

  • 确定去往目的网络的最佳路径：从源到目的的最小费用路径。

链路的权值表示费用，c(x,x’)

路由算法分类

  • 静态路由（手工配置、更新慢、优先级高）、动态路由（路由更新快）
  • 全局（所有路由器掌握完整的网络拓扑信息）、分散（路由器只掌握物理相邻的邻居及链路费用）

链路状态算法

Dijkstra

  • 所有结点掌握网络拓扑和链路费用
    • 通过链路状态广播（泛洪）
    • 所有结点拥有相同信息
  • 计算一个结点到达其他结点的最短路径（获得转发表）
  • c(x,y)：结点x到结点y的链路费用；D(v)：从源到目的v的当前路径费用值；p(v)：从源到v的当前路径，v的前序结点；N’：已经找到最小费用路径的结点集合。

存在震荡（oscillation）的可能

距离向量算法

Bellman-ford

  • dx(y)：从x到y最短路径的费用。dx(y)=min{c(x,v), dv(y)}，v为x的邻居
  • Dx(y)：从x到y的最小费用估计。Dx(y)=[Dx(y): y$\in$N]，N为所有点的集合
  • 每个结点维护所有邻居的距离向量：Dv=[Dv(y): y$\in$N]
  • 结点获得最短路径的下一跳，该信息用于转发表中
  • 核心思想：每个结点不定时地将自身的Dv估计发给其邻居。当x接到邻居的Dv时，依据BF更新其自身的距离向量估计。
  • 异步迭代（局部迭代）、分布式（仅当Dv变化才通知邻居）

好消息传播块，坏消息传播慢。

若某一结点链路发送变化，相邻结点感知速度慢，可能产生无穷计数问题。

解决办法：毒性逆转。

  • 毒性逆转：若一个结点到达某目的的最小费用是通过某个邻居，则告知该邻居节点到达目的的距离为无穷大。

层次路由

将任意规模的网络抽象为一个图计算路由，过于理想化。考虑到网络规模、管理自治两个因素，可以分层。

  • 聚合路由器为一个区域：自治系统AS（autonomous systems）

  • 同一AS内的路由器运行相同的路由协议（算法）

    • 自治系统内部协议。“intra-AS” routing protocol

  • 网关路由器（gateway router）

    • 位于AS边缘，通过链路连接其他AS的网关路由器。

  • 转发表由AS内部路由算法与AS间路由算法共同配置。

    • 若能通过多个AS到达目的，则根据热土豆路由原则，即将分组发送给最近（费用最低）的网关路由器。

OSPF协议（自治系统内）

OSPF：open shortest path first，开放最短路径优先协议。

OSPF封装在IP报文中，RIP封装在UDP报文中。

  • 自治系统内部协议
  • 使用链路状态路由算法
  • OSPF通告中每个入口对应一个邻居
  • OSPF通告在整个AS范围泛洪
    • OSPF报文直接封装在IP数据报中

优点：

  • 安全：所有OSPF报文可被认证
  • 允许使用多条相同费用的路径（RIP只能选择一条）
  • 对于每条链路，可以针对不同的TOS设置多个不同的费用度量（TOS为首部中的服务类型）
  • OSPF支持对大规模AS再次分层

分层的OSPF

  • 两级分层：主干区（backbone）、局部区（local area）
  • 链路状态通告只限于区域内
  • 每个路由器掌握所在区内的详细拓扑
  • 只知道去往其他区网络的“方向”（不知详细拓扑）
  • 区域边界路由器（area border router），既是主干区路由器，也是局部区路由器
  • AS边界路由器（boundary router），负责连接其他自治系统

BGP协议（自治系统间）

BGP：border gateway protocol，边界网关协议

  • 自治系统间协议（标准区域间路由协议）
  • eBGP：从邻居AS获取子网可达性信息
  • iBGP：向所有AS内部路由器传播子网可达性信息
  • 基于可达性信息与策略，确定到达其他网络的路径

BGP会话（session）：两个BGP路由器交换BGP报文。

  • 通告去往不同目的前缀（prefix）的路径
  • 报文交换基于半永久的TCP连接
  • 前缀+属性（AS路径，下一跳）=路由

BGP报文：

  • OPEN：与另一个BGP建立TCP连接，并认证对方
  • UPDATE：通告新路径（或撤销原路径）
  • KEEPALIVE：在无UPDATE时，保活连接；也用于对OPEN请求的确认
  • NOTIFICATION：报告先前报文的差错，也用于关闭连接

BGP路由选择：

  • 基于策略（policy）路由（本地偏好值、最短AS-PATH、热土豆路由等）

BGP路由选择策略：

  • 不同自治系统之间的通告存在策略。

  • 提供商网络、客户网络、桩网络（只与一个AS连接）、双宿网络（与两个AS连接）

  • 提供商只为客户路由

为什么采用不同的AS内与AS间路由协议？

  1. AS间：期望能够管理控制流量如何被路由，谁路由经过其网络等
  2. AS内：单一管理，无需策略决策
  3. 层次化节省路由表大小，减少路由流量更新，适应大规模网络

ICMP协议

ICMP：互联网控制报文协议

用于报告主机或路由器异常、差错。

ICMP 报文封装到IP数据报中传输。

链路层

数据传输形式：帧（frame）

帧首部中的“MAC”地址，用于标识帧的源和目的

链路层在“适配器”中实现，即网卡

差错检测

  • 奇偶校验（1比特位、二维）
  • 检验和
  • 循环冗余校验码（CRC）
    • 检测能力强
    • 设数据比特，D（二进制表示）
    • 设r+1位的生成比特模式，G
    • 目标：选择r位的CRC比特，R满足：
      • <D,R>刚好可以被G整除（模2）
      • 检测：利用<D,R>除以G，余式全0则无措，否则有错
      • 可以检测所有突发长度小于r+1位的错
    • 令 $D*2^r xor R=nG$
    • 所以，R的求法为:$R=(D*2^r)/G$
    • 除法为异或的形式

多路访问控制协议

multiple access control protocol（MAC）

链路类型：

  1. 点对点
  2. 广播

冲突：单一共享广播信道，两个或两个以上的结点同时传输

MAC协议分类：

  1. 信道划分。多路复用技术，如TDMA、FDMA

  2. 随机访问。信道不划分，允许冲突，采用冲突“恢复”机制，如ALOHA、CSMA、CSMA/CD

  3. 轮转。如令牌环网，polling

信道划分

TDMA: time division multiple access

  • “周期性”接入信道，每个站点在每个周期，占用固定长度的时隙

FDMA: frequency division multiple access

  • 信道频谱划分为若干频带，每个站点分配一个固定的频带

随机访问

当结点要发送分组时，利用信道全部数据速率R发送分组。若两个或多个结点同时传输，则发生冲突，随机访问协议需要检测冲突，并从冲突中“恢复”。

时隙ALOHA

假定：所有帧大小相同；时间被划分为等长的时隙；每个时隙可传输1个帧；结点只能在时隙开始时刻发送帧；结点间时钟同步；如果2个或2个以上结点在同一时隙发送帧，结点即检测到冲突。

运行：当结点有新的帧时，在下一个时隙(slot)发送。如果无冲突：该结点可以在下一个时隙继续发送新的帧；如果冲突：该结点在下一个时隙以概率p重传该帧，直至成功。

最大效率：37%

  • 非时隙(纯)Aloha：更加简单，无需同步，当有新的帧生成时，立即发送，但冲突可能性增大。

    • 最大效率：18%

CSMA

载波监听多路访问协议CSMA (carrier sense multiple access)

  • 发送帧之前，监听信道(载波)：信道空闲，发送完整帧；信道忙，推迟发送。
  • 冲突可能仍然发生：信号传播延迟。

为了解决信号传播延迟的问题，可以改进为CSMA/CD协议。

  • 边发送信号时边检测，不发送信号时不检测。

  • 短时间内可以检测到冲突，冲突后传输中止，减少信道浪费。

  • 有线局域网易于实现：测量信号强度，比较发射信号与接收信号。

  • 无线局域网很难实现：接收信号强度淹没在本地发射信号强度下。

  • 效率：$\frac{1}{1+\frac{5t_prop}{t_trans}}$，其中$t_prop$表示LAN中两个结点的最大传播延迟，$t_trans$表示最长帧传输延迟

    CSMA/CD的具体传输过程如下：

  1. NIC（网卡）监听信道：如果监听到信道空闲，则开始发送帧；如果NIC监听到信道忙，则一直等待到信道空闲，然后发送帧。
  2. NIC发送完整个帧，而没有检测到其他结点的数据发送，则NIC确认帧发送成功！
  3. 如果NIC检测到其他结点传输数据，则中止发送，并发送堵塞信号。
  4. 中止发送后，NIC进入二进制指数退避：
    • 第m次连续冲突后：
      • 取n = Min(m, 10)
      • 从{$0,1,2,\dots,2^n-1$}中随机选择一个数为K
      • 等待K·512比特的传输延迟时间，再返回第1步
    • 连续冲突次数越多，平均等待时间越长

局域网

IP地址：32位。接口的网络层地址，用于标识网络层(第3层)分组，支持分组转发。

MAC地址：48位。用于局域网内标识一个帧从哪个接口发出，到达哪个物理相连的其他接口，固化在网卡中。

  • 局域网中的每块网卡都有一个唯一的MAC 地址。

  • MAC地址由IEEE统一管理与分配，网卡生产商购买MAC地址空间(前24比特)。

  • 类比：IP地址为邮政地址。MAC地址为身份证号。

ARP协议

如何在已知目的接口的IP地址址前提下确定其MAC 地址？使用ARP表

  • LAN中的每个IP结点(主机、路由器)维护一个ARP表。
  • 存储某些LAN结点的IP/MAC地址映射关系:< IP地址; MAC地址; TTL>
  • TTL (Time To Live)：经过这个时间以后该映射关系会被遗弃(典型值为20min)

同一局域网内的ARP协议运作：

  1. A想要给同一局域网内的B发送数据报。若B的MAC地址在A的ARP表中，直接取出表中IP对应的MAC地址进行转发；若B的MAC地址不在A的ARP 表中，进行第2步

  2. A广播ARP查询分组，其中包含B的IP地址

    • 目的MAC地址 = FF-FF-FF-FF-FF-FF
    • LAN中所有结点都会接收ARP查询

  4. B接收ARP查询分组，IP地址匹配成功，向A应答B的MAC 地址

    • 利用单播帧向A发送应答

  6. A在其ARP表中，缓存B的IP-MAC地址对，直至超时。

  • 若传输时跨越了局域网，则A向路由器网关发送数据报，通过路由器来转发到B。到达路由器后，数据帧中MAC地址会改变。

以太网

ETHERNET：应用最广泛的有线LAN技术，造价低廉，满足网络速率需求。

类型：

  1. 总线型bus（所有结点在同一冲突域）
  2. 星型star（中心交换机，每个结点一个单独冲突域）

特点：无连接，不可靠

  • 以太网的MAC协议：采用二进制指数退避算法的CSMA/CD

以太网帧的结构：

  • 前导码：用于发送端与接收端的时钟同步
  • 目的MAC地址、源MAC地址（若目的MAC地址为广播地址，FF-FF-FF-FF-FF-FF）
  • 类型(2B)：指示帧中封装的是哪种高层协议的分组，如IP
  • 数据(Data)(46-1500B)：指上层协议载荷，MTU=1500
    • 存在数据最小值，是考虑到CSMA/CD发送数据存在时延，若数据太小，则发送太快，检测不到是否冲突
  • CRC(4B)：循环冗余校验码

以太网的标准不同，导致速率不同，介质不同（光纤，铜缆）

交换机

  • 链路层设备，存储、转发以太网帧，选择性向一个或多个输出链路转发帧
  • 自学习，无需配置
  • 即插即用
  • 利用CSMA/CD访问链路，发送帧，每段链路一个独立的冲突域

交换机如何知道从主机A到主机B？

  • 每个交换机有一个交换表（主机的MAC地址, 到达主机的接口, 时间戳）
  • 交换机通过自学习，获知到达主机的接口信息
    • 当收到帧时，交换机“学习”到发送帧的主机（通过帧的源MAC地址），位于收到该帧的接口所连接的LAN网段。将发送主机MAC地址/接口信息记录到交换表中。

当交换机收到帧：

  1. 记录帧的源MAC地址与输入链路接口
  2. 利用目的MAC地址检索交换表
  3. if 在交换表中检索到与目的MAC地址匹配的入口(entry)
    then {
    if 目的主机位于收到帧的网段
    then 丢弃帧
    else 将帧转发到该入口指向的接口
    }
    else 泛洪(flood) //向除收到该帧的接口之外的所有接口转发

虚拟局域网VLAN

若局域网为单一广播域，所有第2层广播流量(ARP,DHCP, 未知目的MAC地址位置)必须穿越整个LAN。

支持VLAN 划分的交换机，可以在一个物理LAN 架构上配置、定义多个VLAN。

划分VLAN的作用：

  • 流量隔离
  • 动态成员：端口可以动态分配给不同VLAN

中继端口(trunk port)：在跨越多个物理交换机定义的VLAN承载帧

  • 为多VLAN转发802.1帧容易产生歧义 (必须携带VLAN ID信息)
  • 802.1q协议为经过中继端口转发的帧增加/去除额外的首部域
